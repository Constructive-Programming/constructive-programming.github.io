Constructive Programming is a methodology for building complex software inspired from constructive logic.

It leverages the the curry Howard lambec equivalence to arbitrarily scale software that is formally verifiable from the way it is written. 

Constructive Programming is about the restrictions you can impose on your code, in order to make it easier to understand, maintain, and evolve.

There are three starting criteria for constructive programs:

- Referential Transparency
- Totality
- Termination
